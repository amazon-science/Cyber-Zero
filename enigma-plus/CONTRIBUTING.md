# Contribution guidelines
The easiest way to contribute to EnIGMA+ is to give us feedback.

* **Something isn't working?** Open a [bug report](https://github.com/amazon-science/enigma-plus/issues/new?template=bug_report.yml).
  Rule of thumb: If you're running something and you get some error messages, this is the issue type for you.
* **You have a concrete question?** Open a [question issue](https://github.com/amazon-science/enigma-plus/issues/new?template=question.yml).
* **You are missing something?** Open a [feature request issue](https://github.com/amazon-science/enigma-plus/issues/new?template=feature_request.yml)
* **Open-ended discussion?** Talk on [discord](https://discord.gg/AVEFbBn2rH). Note that all actionable items should be an issue though.
## CTF Challenge Contributions

We especially welcome contributions that improve EnIGMA+'s performance on CTF challenges:

* **New CTF categories**: Help us add support for additional CTF challenge types
* **Tool integrations**: Add new cybersecurity tools to the container environment
* **Interactive Agent Tools**: Improve or add new IATs for CTF solving
* **Performance optimizations**: Enhance parallel execution and efficiency
* **Challenge-specific configurations**: Create tailored configs for specific CTF competitions
* **Evaluation improvements**: Contribute to fair comparison frameworks and benchmarking

## Cyber-Zero Project

EnIGMA+ is part of the broader [Cyber-Zero](https://amazon-science.github.io/cyber-zero/) project. Contributions that align with Cyber-Zero's mission of advancing AI for cybersecurity are particularly welcome.

You want to contribute to the development? Great! Please see the [development guidelines](https://amazon-science.github.io/cyber-zero/dev/contribute/) for guidelines and tips specific to CTF agent development and the Cyber-Zero ecosystem.
